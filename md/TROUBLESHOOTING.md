# 🔧 Решение проблем (Troubleshooting)

### 1. Сайт не открывается (502 Bad Gateway)
Это значит, что Nginx работает, но Python-бот (FastAPI) упал или не запущен.

**Решение:**
1. Зайдите на сервер.
2. Проверьте логи PM2: `pm2 logs CatDock`.
3. Если бот упал, перезапустите: `pm2 restart CatDock`.

### 2. Изменения на сайте не видны
Вы правите файлы в `/root/Bots/CatDock/web`, но на сайте старая версия.

**Причина:** Nginx берет файлы из `/var/www/CatDock`.
**Решение:** Запустите скрипт деплоя:
```bash
./deploy_web.sh
```
И сбросьте кэш браузера (Ctrl+F5).

### 3. Ошибка подключения к базе данных
В логах: `asyncpg.exceptions.CannotConnectNowError` или `Connection refused`.

**Решение:**
1. Проверьте, жив ли контейнер БД: `docker ps | grep postgres`.
2. Если нет, поднимите его: `docker compose up -d`.

### 4. Ошибки миграций (Alembic)
Если при запуске `alembic upgrade head` вы видите ошибку `relation "users" already exists`, значит база уже создана старым способом.

**Решение:**
Синхронизируйте Alembic с текущим состоянием базы (пометить как "актуальную"):
```bash
alembic stamp head
```

### 5. Контейнеры пользователей не создаются
В логах: `SSH connection failed`.

**Решение:**
1. Проверьте доступность ноды (сервера) по SSH вручную.
2. Проверьте баланс на серверах (если это VPS).
3. Проверьте, не переполнен ли диск на ноде: `df -h`.

### 6. Ошибка 404 на страницах типа /profile
Если при обновлении страницы `/profile` вы видите 404 от Nginx.

**Причина:** Не настроен `try_files` в конфиге Nginx.
**Решение:** Проверьте `md/NGINX_CONFIG.md` и убедитесь, что в конфиге есть строка:
```nginx
location / {
    proxy_pass http://127.0.0.1:8082;
    # ...
}
```
Или, если статика настроена отдельно, убедитесь, что SPA-роутинг работает.
